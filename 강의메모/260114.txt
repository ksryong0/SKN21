MVT

함수 내부 구조
1. 입력파라미터 검증 => View
2. Business Logic : 실제 수행할 작업 => View?
db 로직 => Model
3. 결과를 출력 -> 처리 결과를 사용자에게 보여주는 것. => Template

출력결과를 밖으로 빼자.
business logic 중 반드시 들어가는게 db 로직임. db 작업.

MVC에서는
나누기, db 로직을 model
입력파라미터검증을 controller
결과출력을 view

MVT패턴의 장점
뭐 하나 틀렸을 때, 틀린 부분만 고치면 됨. 유지보수도 좋음. 재사용성도 좋음.

mvt 패턴의 종류는 클래스 기반과 함수 기반이 있음.

boiled template : mvt 패턴에 따라 틀을 만들어줌.
