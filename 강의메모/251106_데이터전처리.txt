데이터 전처리 : 내가 하려는 일에 맞는 데이터만 남겨놓는 것
여기서 전이란 모델링 전.

결측치 처리방법
	제거 : 열 제거보다는 행 제거가 더 많음
		결측치가 있는 행 삭제
		결측치가 있는 열 삭제
	대체 : 가능성이 높은 값으로 대체
		평균, 중앙값, 최빈
		예측모델
			KNN : 결측치 근처 값들을 모아서 평균, 중앙, 최빈값으로 대체
	도메인 지식 : 경험이 많은 사람들의 도움?

이상치(Outlier) 처리
	잘못들어간 값(오류) : 나이에 -99. 혈액형에 K. 
	극단치(정상범위 넘어간 값) : 중학생이 20살. 데이터의 일반화를 해친다(떨어트림).

분포에서 벗어난 이상치 식별
	도메인 기준 : 경험에 의한. 계산해서 알 수 없는 것들.
			ex)보통 중학생은 20살이 아니야.
	통계적 기준 : 계산해서 찾아냄.
		표준편차기준 : 데이터가 정규분포를 따를 때.  
			예)-2표준편차~2표준편차에 95% 값이 들어가면, 그 이외는 이상치
			-3표준편차~3표준편차에 98% 값이 들어가면, 그 이외는 이상치

			평균-k*표준편차 <= 값 <= 평균+k*표준편차 
			(k: 몇분위)
		분위수 기준(IQR 이용): 평균이 왜곡될 때.
			IQR = 3분위-1분위
			1분위-whis*IQR <= 값 <= 1분위+whis*IQR

극단치 처리
	제거 : 예)앞으로는 20살 중학생은 없을거다 하면 제거
	윈저화 : 범위를 정함. 정해둔 최소값보다 작으면 최소값으로 대체. 정해둔 최대값보다 크면 그 최대값으로 대체.
	대체 : 평균,중앙,최빈 값으로 대체

feature별 전처리 : 모델의 성능을 높여줌
	범주형변수
		명목/비서열 변수
		순위/서열 변수
	수치형변수:수량을 세는
		이산형:하나 두개 세개 네개
		연속형:실수형.

범주형 데이터 전처리:문자열은 사용 불가. 숫자로 바꿔줘야 함. 사이킷 런은 문자열을 못다룸. 다른 모듈은 문자열을 숫자로 바꿔서 씀.
	레이블(label) 인코딩 : 강의자료 그림 참고. TV는 0, 냉장고는 1 이런식으로 라벨을 붙임. y값이 숫자고 x값이 문자열일 때. 
	트리계열 모델에 적용. 예)디시젼 트리 모델에 적용
	선형 계열 모델에 적용하면 안됨 -> TV와 냉장고 에어컨은 1의 차이가 있는게 아니고, 단순히 라벨을 붙인거임. 그래서 x가 선형구조가 아니니까 사용하면 안됨.
	레이블인코딩 관련 사이킷런 전처리함수 : fit함수는 뭐를 뭐로 바꿀지 학습. transform으로 변환.
	원핫인코딩 : N개의 클래스를 N차원의 원핫벡터로 변환. 
			예) TV니? 맞으면 1이고 나머지는 0.
			상품분류라는 컬럼 1개가 상품의 종류 수에 따라 늘어남.
			이런 1이 적고, 0이 많은건 Sparse Matrix라고 하는데, 디시전 트리 모델에서는 잘 예측을 못함.
			사이킷런 함수:
			fit
			transform
			fit_transform
			get_feature_names_out	

수치형 데이터 전처리 : 
	Feature scaling : 컬럼간에 척도를 맞춰주겠다.
		표준화 scaling : (모든 값들-평균)/표준편차    평균0 표준편차1 안에 있도록 함. fit할떄 평균과 표준편차 찾음. transform할떄 식 대로 변환
		min max scaling : (모든값들-최소값)/(최대값-최소값)   모든 값을 최소값 0~최대값1로 변환. fit할떄 최대값, 최소값 찾음. transform할떄 식 대로 변환

train_set 기준으로 전처리를 해야함. 그래야 모델의 성능을 제대로 판단할 수 있음.

모델평가
	분류:얼마나 잘 맞추는지
		이진분류 : Positive(1) or Negative(0). 환자? 스팸메일? 사기거래? 
		다중분류 
	분류 지표(기본적으로 이진분류용 분류 지표)
		정확도 : 맞게예측한건수/전체예측건수. accuracy_score 함수 이용. 정확도만으로는 평가가 정확하게 될 수 없다.
		정밀도 : 
		재현률
		F1 점수
		PR Curve, AP score
		ROC, AUC score
	회귀:정답이 될수있는 애들이 무한대. 정확하게 알 수 없음.
		MSE
		RMSE
		R2
sklearn.metrics (정답, 추정값)

혼동 행렬
0번축:정답
1번축:예측
True 붙으면 맞은거.
False붙으면 틀린거.
		예측
		0	1
	0	TN	FP
실제	1	FN	TP

F1점수:Recall과 Precision을 같이 봄
recall : Positive 값 (실제 정답)을 얼마나 잘 맞추냐.
precision : TP 비율. 정답으로 예측한 것 중에, 실제로 맞춘 비율.

recall과 precision은 기억해야함.
accuray:전체중에 맞은거(틀린지 맞는지 예측이 맞음)
공식은 기억못해도 뭘 의미하는지는 알고 있어야함.
